### *0x00 引言*
&emsp;&emsp; 很多时候需要自己安装操作系统，比如重装系统、装双系统等等情况，这里将记录操作系统安装的相关内容。

### *0x01 自制系统U盘*
&emsp;&emsp;使用U盘制作系统盘是种廉价且方便的方法，网上有许多教程，具体方法可分为两种，第一种是[PE安装](https://jingyan.baidu.com/article/ab69b2706bbb1a2ca7189f2d.html)<sup>[1]</sup>，第二种是[ISO写入](https://jingyan.baidu.com/article/5225f26b0bb45fe6fa0908bc.html)。前者更加可靠，且系统U盘仍有一部分可以当作U盘来使用，且一个U盘可以放多种系统。
```
1 PE制作系统U盘
  1.1 微PE制作Windows系统U盘
  1.2 微PE+FbinstTool制作多系统U盘
2 ISO烧制系统U盘
```

***1、PE制作系统U盘***
***1.1、微PE安装Win***
&emsp;&emsp;PE(Windows Preinstallation Environment)，是Windows的预安装环境，可以理解为一种占用很小存储量的系统。由于本身就是Windows的原装系统，所以实际上只是对Win系统的安装友好，这里以安装Win10系统为例，安装过程可查阅[百度经验](https://jingyan.baidu.com/article/3aed632eda312b701080910b.html)。
&emsp;&emsp;1)安装PE工具，并用其格式化U盘。PE工具优点在于，体积小但是功能齐全，没有植入广告和垃圾软件，下载链接在上述经验<sup>[1]</sup>里。
&emsp;&emsp;2)从[阿里云镜像](http://mirrors.aliyun.com/ubuntu-releases/14.04/)<sup>[2]</sup>上下载对应的.iso文件，并将其放入U盘。
&emsp;&emsp;OK！系统U盘已制作完成，接下去可以选择依赖于U盘的单个系统重装或是双系统安装。
&emsp;&emsp;这里有几点需要注意:1)微PE格式化U盘以后生成两个盘符，其中一个用来存放PE系统，另一个较大的盘用来存放想安装的系统ISO或者GHOST，两者均可在PE系统桌面下直接安装;2)可以直接去PE系统桌面下进行盘符分区，也可以直接在旧的系统下分区(这里只是方便没有系统的电脑分区)。
***1.1、PE+FbinstTool制作多系统U盘***
&emsp;&emsp;由于PE系统桌面本身是无法直接安装Ubuntu的，因此我们需要借用另一个工具-FbinstTool。该工具实际上与微PE类似，只不过实际操作过程有点区别，我们只需要安装一遍就能理解。这里以4G内存U盘PE方法制作Ubuntu14.04为例，具体的操作过程可参见为数不多的一篇[博文](https://www.cnblogs.com/acbingo/p/4515240.html)，我们用到的工具实质上为微PE+FbinstTool，前者用来格式化U盘并加入一个基础的PE系统(与上一节一致)，而后者就可以直接放在U盘下依靠一个名为grldr(类似于Cmakelist)的文件被系统识别，它负责告诉系统我的U盘里包含了哪些系统。根据博客里的指导，我们先看下U盘非PE盘符下的文件结构：

&emsp;&emsp;可知，我把Ubuntu14.04的ISO文件放在了目录G:/ISO下，另外两个文件initrd与vmlinuz.efi是从镜像文件里拿出来的，具体位置看博客，原因似乎是FbinstTool需要调用，那么接下来看看FbinstTool的配置文件grldr究竟在说些什么：
```
#不用管这两行了
default 0
timeout 60
#Auto方式启动PE，启动PE的方式也不用管了
title Windows PE (Auto)
find --set-root --ignore-floppies /wepe/wepemenu.ini || find --set-root --ignore-floppies /wepe.iso
configfile /wepe/wepemenu.ini || map /wepe.iso (0xff) && map --hook && chainloader (0xff)
#EFI方式启动PE，可删
title Windows PE (From EFI)
errorcheck off
find --set-root --ignore-floppies /wepe/wepemenu.ini && configfile /wepe/wepemenu.ini
errorcheck on
#UD方式启动PE，可删
title Windows PE (From UD)
errorcheck off
map (ud)/wepe.iso (0xff)
map --hook
chainloader (0xff)
errorcheck on
#关键在这儿，引入名为Ubuntu14.04的系统，且指明重要文件的位置
title Ubuntu14.04
find --set-root /ISO/ubuntu-14.04.6-desktop-amd64.iso
kernel /ISO/vmlinuz.efi boot=casper iso-scan/filename=/ISO/ubuntu-14.04.6-desktop-amd64.iso
initrd /ISO/initrd
boot
#为了制作多系统U盘可以仿照上面一条的方法自行添加
```

***3、任意系统重装***
***3.1、使用微PE+FbinstTool制作的多系统U盘***
&emsp;&emsp;(1)直接U盘方式启动，先运行Auto PE，进入PE系统后设置盘符，由于安装单系统，所以格式化所有盘符;
&emsp;&emsp;(2)重启，并以title名为 Ubuntu14.04 U盘进入方法启动，可以看到进入了一个Ubuntu系统桌面，但是桌面上还有两个文件，其中一个是Install Ubuntu14.04，按步骤安装即可;
&emsp;&emsp;(3)安装完成，重启进入BIOS，并将U盘启动后移，继续重启即可运行Ubuntu系统。

***4、Ubuntu系统相关***
***4.1、Ubuntu系统磁盘分配推荐***  
&emsp;&emsp;磁盘分配即系统使用过程中硬盘空间的资源合理配置，这里记录一下比较适合的[分配方法](https://www.cnblogs.com/116970u/p/10487452.html)，下面以128G工控机为例。
&emsp;&emsp;首先，介绍各挂载点作用：

| 挂载点 | 分区类型    | 说明                      |
| ----- | -------- | ------------------------------------------------- |
| /     | 主分区 | 用于存放系统，相当于windows的C盘，15G即可 |
| /home | 逻辑分区 | 用户存储数据用，要尽可能大           |
| /boot | 主分区 | 引导分区，大小设置为480M                |
| swap  | 逻辑分区 | 相当于电脑内存，大小设置为电脑内存大小(2/4/8/16G)|

&emsp;&emsp;给出高级用户Desktop版本的分配建议：

| 挂载点 | 装置    | 说明                      |
| ------ | --------- | --------------------------- |
| /      | /dev/hda1 | 15G                         |
| /home  | /dev/hda2 | 最大的剩余空间       |
| /boot  | /dev/hda3 | 480MB左右                 |
| swap   | /dev/hda5 | 大约内存大小(建议至少512MB) |

&emsp;&emsp;高级用户Server版本的分配建议：

| 挂载点 | 装置    | 说明                      |
| ------ | --------- | --------------------------- |
| /      | /dev/hda1 | 15G                         |
| /home  | /dev/hda2 | 最大的剩余空间       |
| /boot  | /dev/hda3 | 480MB左右                 |
| swap   | /dev/hda5 | 大约内存大小(建议至少512MB) |
| /var   | /dev/hda6 | 视服务器功能决定大小 |

### *0x06 Docker容器技术*
&emsp;&emsp;Docker容器技术与虚拟机一样，是在当前系统下使用其他系统软件的方法，但它更方便快捷，占用体积更小，运行目标程序只需要拥有一个配置好环境的镜像文件即可。关于Docker可以参考[教程](https://www.runoob.com/docker/docker-hello-world.html)。另外比较重要的一点是Docker所使用的[四种网络连接方式](https://blog.51cto.com/11880730/2115614)，比较幸运的是其默认使用桥接模式，方便信息安全方面的技术研究。
&emsp;&emsp;如教程所示，Win10专业版有较方便的安装方法，但是普通版本的Win10安装Docker还有许多坑在那儿。这里讲一讲Win10安装Docker的步骤
&emsp;&emsp;1)从[阿里云镜像](http://mirrors.aliyun.com/docker-toolbox/windows/docker-toolbox/)种下载最新的Docker toolbox，这个ce版本在Win10上是支持的。
&emsp;&emsp;2)安装下载文件，git可以选用自己以前安装的，所以不用下载，只需在安装完成后，右键Docker Quickstart Terminal属性，并修改目标git的bash.exe路径即可。
&emsp;&emsp;3)右键管理员方式启动Docker Quickstart Terminal，发现程序一直[卡死在boot2docker.iso的下载](https://blog.csdn.net/dhaiuda/article/details/81975550)，只需按链接里的方法即可解决，原因就是没有翻墙，也可以ss开启全局模式下载。
&emsp;&emsp;4)每次运行都需要右键管理员模式运行。

### *0x07 软件安装脚本与镜像制作*
